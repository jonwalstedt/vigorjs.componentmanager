Filter components by string matching

filterString on the instanceDefintion

These examples have the filterString defined on the instanceDefinitions and sets the properties "includeIfStringMatches", "hasToMatchString" or "cantMatchString" on the passed filter. This approach allows for an easy and convenient way of grouping components using the filterString property on the instanceDefinition (ex. using the property for grouping by type)

  • Restore
  • includeIfStringMatches "test"
  • hasToMatchString "test"
  • cantMatchString "test"

filterString on the filter

These examples have the filterString defined on the filter passed to the refresh method. This approach is pretty much the opposite of the previous example - here the properties "filterStringHasToMatch" and "filterStringCantMatch" are set on the instanceDefinition. This allows for an easy way to filter instances on a common string, ex. a string containing the application state, if you want to show certain components for a specific language or anything else that is not necessarily contained in the url.

  • Restore
  • filterStringHasToMatch "test"
  • filterStringCantMatch "test"

Take a look at components.js to see component the setup for this example
